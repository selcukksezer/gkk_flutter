## Notes / caveats on the simulation model itself

1. **Item sink is non-functional in the model.** `qa_sim_daily_events.items_burned` sums to **0** for the entire run despite the per-day formula `floor((market_sells + craft_actions) * 0.55)`. Either the column update is being overwritten or the formula evaluates to 0 — this makes the 100% item-inflation figure a **model artifact, not a measured economy result**. Fix the sim before trusting item inflation.
2. **Exploit "battery" is 6 types, not the planned 50 (E01–E50).** Several are derived from telemetry aggregates rather than true RPC race attacks (`cooldown_bypass` only flags `dungeon_runs>=4`; it never replays `attack_dungeon` inside a cooldown window). `premium_stack_abuse` IS a real live replay (`buy_vip_pass` called twice) and its 23% success is a genuine **VIP idempotency defect**.
3. **`multi_account_gold_funnel` 88% success** is measured from `pvp_matches.gold_stolen > 300` by multi-segment bots — a real signal that PvP gold transfer between linked accounts is unguarded.
4. Two retention definitions coexist (`checkpoints.retained_users` via `churn_risk` vs. export `active/total`); this report uses the export definition.
